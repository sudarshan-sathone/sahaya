# AidChain — Project Documentation
## Prayatna 3.0 Hackathon 2026 | Acropolis Institute of Technology

---

## What We Are Building
A trustless blockchain system where donors send ETH to a smart contract,
NGO workers register verified disaster victims, victims get OTP vouchers,
and vendors redeem those vouchers for funds — with every transaction
publicly visible on-chain. The victim never touches blockchain.

## Problem
Disaster relief funds pass through 6 middlemen before reaching victims.
Money leaks at every layer. Donors have zero visibility. Victims wait months.

## Solution
Replace all middlemen with a smart contract that holds funds and releases
them only when verified conditions are met — enforced by code, not trust.

## Judge Requirements Addressed
1. Regular transaction monitoring — TransactionMonitor component with
   auto-refresh, suspicious activity flags, full transaction history
2. Admin dashboard — AdminDashboard with real-time stats, campaign health,
   NGO activity scores, fund flow overview
3. Strict NGO monitoring — NGO registration, verification, activity scoring,
   redemption rate tracking, suspension capability
4. Campaign authenticity — document hash required, Pending→Verified→Active
   status flow, authenticity badges, flagging for inactive campaigns
5. Data analysis — AnalyticsDashboard with charts, historical trends,
   exportable training data, recharts visualizations
6. ML prediction — PredictionPanel estimating disaster cost and relief
   duration from historical data, ML API integration ready

## Disaster Types (contract enum)
0=Flood, 1=Earthquake, 2=Cyclone, 3=Drought, 4=Fire, 5=Pandemic, 6=Other

## Campaign Status Flow (contract enum)
Pending (0) → Verified (1) → Active (2) → Flagged (3) → Closed (4)

## NGO Status (contract enum)
Pending (0) → Active (1) / Suspended (2)
